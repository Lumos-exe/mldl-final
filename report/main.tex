\begin{abstract}
卷积神经网络具有局部感受野和平移等变等归纳偏置，通常能在中小规模图像数据集上稳定训练；视觉 Transformer 依靠自注意力机制建模长程依赖，但在小数据和中等参数预算下容易出现优化困难或过拟合。本文围绕 CIFAR-100 图像分类任务，实现轻量 CNN、轻量 ResNet-Transformer Hybrid、CIFAR 版 ResNet-18、强 Hybrid 和 Small ViT，并通过严格消融实验研究轻量 Hybrid 中 attention 机制、Transformer 层数和 patch size 的影响。实验重点不追求大型模型复现，而是在统一训练与评估协议下比较不同参数规模下 CNN 与 CNN-Transformer 混合结构的差异。
\end{abstract}

\noindent\textbf{关键词：} CIFAR-100；ResNet；视觉 Transformer；混合网络；消融实验

\section{引言}
CNN 通过卷积核共享和局部连接高效提取边缘、纹理和局部形状等视觉特征，是小图像分类任务中的强基线。ViT 将图像切分为 token 并使用多头自注意力建模 token 间关系，具备更灵活的全局依赖建模能力，但其缺少 CNN 的局部归纳偏置，通常需要更多数据、正则化和训练技巧。已有混合视觉网络并不是简单地在 CNN 后面追加 Transformer：MobileViT\cite{mehta2022mobilevit} 将 Transformer 作为卷积特征中的全局表示模块，形成局部--全局--局部的融合流程；CvT\cite{wu2021cvt} 将卷积引入 token embedding 和 Transformer 投影；CoAtNet\cite{dai2021coatnet} 则强调前期卷积与后期 attention 的有序组合；MoCoViT\cite{ma2022mocovit} 和 CMT\cite{guo2021cmt} 也从移动端和高效建模角度说明卷积与自注意力需要在结构中深度融合，而不是只在分类头前串接。

本文选择 CIFAR-100\cite{krizhevsky2009learning} 作为主任务。该数据集图像分辨率仅为 $32\times32$，但包含 100 个类别，比 CIFAR-10 更能体现模型表示能力差异。本文设置两组参数预算接近的对比：轻量组比较约 2.8M 参数的 CNN 与 Hybrid，强组比较约 11M 参数的 ResNet-18 与 Hybrid，并加入 Small ViT 作为纯 Transformer 对照。实验回答两个问题：第一，Transformer 融合在轻量容量下是否比强 CNN 容量下更容易体现收益；第二，在轻量 Hybrid 中，attention 交互、Transformer 层数和 patch size 分别如何影响性能。

\section{方法}
\subsection{CNN baseline}
CNN baseline 采用适配 CIFAR 分辨率的 ResNet-18。模型使用 $3\times3$ 卷积作为 stem，不使用 ImageNet 版 ResNet 的大步长 stem 和 max pooling，以保留 $32\times32$ 小图像的空间细节。本文使用 32 宽 ResNet 作为轻量 CNN baseline，使用 64 宽 ResNet-18 作为强 CNN baseline，从而观察不同容量下混合结构的相对收益。

\subsection{Small ViT}
Small ViT 参考 ViT\cite{dosovitskiy2021image} 的基本思想，将 $32\times32$ 图像划分为固定大小 patch，经卷积式 patch embedding 映射为 token 序列，加入 class token 和位置编码后输入 Transformer Encoder。该模型作为纯 Transformer 对照，用于观察在 CIFAR-100 这种小分辨率数据集上，缺少卷积前端时模型的优化和泛化表现。

\subsection{ResNet-Transformer 混合模型}
混合模型采用较窄的 CIFAR 版 ResNet 主干，以便在接近 CNN baseline 的参数预算下为 Transformer 融合块留出容量。模型在第二阶段输出的 $16\times16$ 特征和第三阶段输出的 $8\times8$ 特征上各插入一个 MobileViT 风格的局部--全局--局部融合块，主配置使用 $2\times2$ patch。具体而言，每个融合块先用局部卷积进一步提炼中层特征，再通过 unfold 操作将特征图组织为 patch token，输入 Transformer Encoder 建模跨 patch 的全局关系；随后通过 fold 操作将 token 无损还原为空间特征图，并通过卷积与原始 CNN 特征融合，最后继续送入后续 ResNet 阶段。相比 stride patch embedding，这种实现避免把 $2\times2$ 局部区域直接压缩为单个 token，更适合保留 CIFAR 小图像中的局部细节。轻量 Hybrid 与轻量 CNN 参数量接近，强 Hybrid 与 ResNet-18 参数量接近，因此可比较不同容量下融合结构的收益。

\section{实验设置}
数据集使用 CIFAR-100，训练集 50,000 张、测试集 10,000 张。本文从训练集中固定划分 5,000 张作为验证集，用于选择最佳 checkpoint；官方测试集只在训练结束后评估一次。训练阶段采用随机裁剪、水平翻转、RandAugment 和标准化；验证和测试阶段仅进行标准化。优化器使用 AdamW，学习率采用 cosine 衰减，损失函数为带 label smoothing 的交叉熵。主实验训练轻量 CNN、轻量 Hybrid、ResNet-18、强 Hybrid 和 Small ViT，并统一报告准确率、参数量与训练曲线；严格消融实验均以轻量 Hybrid 为基准，每次只改变 attention 机制、Transformer 层数或 patch size 中的一个因素。

评价指标包括最终测试集 top-1 accuracy、可训练参数量、验证集 accuracy 曲线和单 epoch 耗时。所有实验固定随机种子，日志保存为 CSV，最佳模型依据验证集准确率保存。

\section{严格消融实验}
消融实验以轻量 Hybrid 为基准，每次只改变一个因素，其他设置保持不变。no-attention 实验保留局部卷积、patch embedding、位置编码、FFN、token 还原、卷积融合和分类头，只将 self-attention 替换为无 token-token 交互的前馈替代模块，用于判断 attention 交互本身是否有效。depth 实验只改变 Transformer Encoder 层数，用于判断增加或减少全局建模深度是否有帮助。patch size 实验只改变特征图切分粒度，用于判断 token 粒度对分类性能的影响。

\section{结论}
本文完成了 CIFAR-100 上轻量 CNN、轻量双阶段 Hybrid、ResNet-18、强双阶段 Hybrid 和 Small ViT 的实现与对比。若轻量 Hybrid 相比轻量 CNN 的提升更明显，而强 Hybrid 相比 ResNet-18 只小幅提升，则说明 Transformer 融合块在容量受限场景下更有价值；若两组提升都有限，则应承认在 CIFAR-100 和当前训练策略下，卷积归纳偏置仍然更划算。后续可进一步尝试更强的数据增强、知识蒸馏或更完整的 MobileViT/CMT 家族结构。
